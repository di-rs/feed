# Contributing Guide

## Adding New Posts

Posts are automatically generated by a cron job, but you can manually add them for testing.

### Post File Structure

Create a new markdown file in: `src/content/posts/{YYYY-MM-DD}/{slug}.md`

Example: `src/content/posts/2026-02-08/my-post.md`

### Frontmatter Template

```yaml
---
title: "Your Post Title"
date: "2026-02-08"           # Required: ISO date format
time: "14:30"                # Required: 24-hour time
source: "Blog Name"          # Required: Source publication
link: "https://example.com"  # Required: URL to original article
image: "https://..."         # Optional: Image URL (leave empty if none)
tags: ["tag1", "tag2"]       # Required: Array of tags (at least one)
---

Your TL;DR content here (2-3 sentences). Keep it concise and informative.
This content will be displayed in the post card.
```

### Validation Rules

- **title**: String, required
- **date**: ISO date string (YYYY-MM-DD), required
- **time**: Time string (HH:MM), required
- **source**: String, required
- **link**: Valid URL, required
- **image**: String (URL), optional - leave empty string if no image
- **tags**: Array of strings, required, at least one tag

### Tag Guidelines

Use lowercase, hyphenated tags:
- ✅ `web-dev`, `machine-learning`, `typescript`
- ❌ `Web Dev`, `Machine_Learning`, `TypeScript`

Common tags:
- `web-dev` - Web development
- `typescript` - TypeScript content
- `ai` - Artificial intelligence
- `machine-learning` - ML/AI
- `performance` - Performance optimization
- `astro` - Astro framework
- `programming` - General programming
- `trends` - Industry trends
- `optimization` - Code/system optimization

### Image Guidelines

- Use high-quality images (min 800px wide)
- Prefer landscape orientation (16:9 or 4:3)
- Use CDN URLs (Unsplash, Cloudinary, etc.)
- Leave empty string if no relevant image
- Test that URLs are accessible

### Content Guidelines

**TL;DR Body:**
- 2-3 sentences maximum
- Summarize the key takeaway
- Write in present tense
- Be objective and informative
- No clickbait or sensationalism

**Example Good TL;DR:**
> Astro's content collections make it easy to build static sites with type-safe frontmatter. Combined with Tailwind CSS, you can create fast, beautiful reading experiences with minimal JavaScript.

**Example Bad TL;DR:**
> You won't believe what this framework can do! Click here to learn the secrets that developers don't want you to know.

## Development Workflow

### Local Development

```bash
# Start dev server
npm run dev

# Open browser to http://localhost:4321

# Watch for changes - auto-reloads
```

### Testing Changes

```bash
# Build the site
npm run build

# Preview production build
npm run preview

# Check for TypeScript errors
npx astro check
```

### Before Committing

1. ✅ Test locally with `npm run dev`
2. ✅ Build successfully with `npm run build`
3. ✅ Check all links work
4. ✅ Verify images load
5. ✅ Test dark/light mode
6. ✅ Test tag filtering
7. ✅ Test date navigation

## Code Style

- Use TypeScript for type safety
- Follow Astro conventions
- Use Tailwind utility classes
- Keep components small and focused
- Comment complex logic
