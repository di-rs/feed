A practical breakdown of the four main application security testing approaches: SAST analyzes source code statically before runtime, DAST tests running apps from the outside, while IAST and RASP operate from inside the application. Each fits different stages of the SDLC with distinct trade-offs between false positives, runtime visibility, and integration points.
