A practical guide to five pitfalls beginners hit when writing Clarity smart contracts on Stacks: forgetting the `.` prefix when calling contracts, mixing signed `int` and unsigned `uint` types, missing Lisp-style parentheses, confusing `let` locals with persistent `define-data-var` storage, and other common syntax traps. Each mistake includes a broken code example and the corrected fix, making it a solid quick-reference for developers coming from JavaScript or Python backgrounds. Using VS Code with the Clarity extension and Clarinet console are recommended for catching these errors early.
